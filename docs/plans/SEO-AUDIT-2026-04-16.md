> **Context:** Show HN launch J-8 (2026-04-24). Audit performed on local prod build (`pnpm build` + `pnpm preview`) since astix.io returns 521 (origin down). Findings reference both `docs/**/*.md` sources and `docs/.vitepress/dist/**/*.html` build artifacts.

## Findings

### 🔴 Critical (blocks indexing or introduces security risk)

#### C1 — Schema.ts plugin broken for blog posts and docs
**Files:** `docs/.vitepress/plugins/schema.ts` L134-195
**Evidence:**
- `dist/blog/introducing-astix.html`: 0 `application/ld+json` blocks (expected BlogPosting).
- `dist/blog/introducing-astix.html`: `og:type: website` (expected `article`).
- `dist/docs/getting-started.html`: 0 JSON-LD blocks (expected TechArticle).
- `dist/index.html`: 3 blocks ✓. `dist/pricing.html`: 2 blocks ✓. `dist/blog/authors/olivier-orabona.html`: 1 block ✓.

**Root cause:** Conditionals check `pageData.relativePath.startsWith('blog/posts/')` but the rewrite `blog/posts/:slug → blog/:slug` changes the relativePath at build time to `blog/X.md`. Similarly `startsWith('guide/') || === 'getting-started.md'` never matches `docs/getting-started.md`.

**Fix (patch):**
```ts
// schema.ts L134 — blog posts
if (pageData.relativePath.startsWith('blog/') &&
    !pageData.relativePath.startsWith('blog/authors/') &&
    pageData.relativePath !== 'blog/index.md') {
  heads.push(['meta', { property: 'og:type', content: 'article' }]);
}

// schema.ts L141 — BlogPosting JSON-LD (same guard)
if (pageData.relativePath.startsWith('blog/') &&
    !pageData.relativePath.startsWith('blog/authors/') &&
    pageData.relativePath !== 'blog/index.md') {
  const fm = pageData.frontmatter;
  if (fm.date) { /* emit BlogPosting */ }
}

// schema.ts L185 — docs as TechArticle
if (pageData.relativePath.startsWith('docs/')) {
  heads.push(jsonLd({ '@context': 'https://schema.org', '@type': 'TechArticle', ... }));
}
```

Impact: 11 blog posts + 3 docs pages (getting-started, mcp-tools, languages) become eligible for Google article rich results, LinkedIn/X article previews, and AI crawler structured-data extraction.

#### C2 — Missing `_headers` file for Cloudflare Pages
**File to create:** `docs/public/_headers`
**Evidence:** `curl -I http://localhost:4173/` returns only `Cache-Control: no-cache`. The site is deployed on Cloudflare Pages (per memory) and has no `_headers` file.

**Fix:**
```
/*
  Strict-Transport-Security: max-age=63072000; includeSubDomains; preload
  X-Content-Type-Options: nosniff
  X-Frame-Options: SAMEORIGIN
  Referrer-Policy: strict-origin-when-cross-origin
  Permissions-Policy: geolocation=(), microphone=(), camera=(), payment=(), usb=(), interest-cohort=()
  Content-Security-Policy: default-src 'self'; script-src 'self' 'unsafe-inline'; style-src 'self' 'unsafe-inline'; img-src 'self' data: https://images.unsplash.com; font-src 'self'; connect-src 'self'; frame-ancestors 'none'; base-uri 'self'; form-action 'self'
  Cross-Origin-Opener-Policy: same-origin
```
Note: `'unsafe-inline'` is required because VitePress inlines styles/scripts. Tighten later via nonces if desired.

#### C3 — Internal planning doc indexed publicly
**File:** `docs/blog-ideas-astix.md`
**Evidence:** Sitemap contains `https://astix.io/blog-ideas-astix` (first entry). `dist/blog-ideas-astix.html` exists with 0 JSON-LD and default description. Content = internal product/SAST/features brainstorm, not publication-ready.

**Fix:** Add to `config.ts`:
```ts
srcExclude: ['superpowers/**', 'blog-ideas-astix.md'],
```
Rebuild and verify URL returns 404.

#### C4 — Stale "10 Tier-1 languages" claim
**Files affected (confirmed):**
- `docs/.vitepress/theme/components/FeatureShowcase.vue` — `desc: '10 Tier-1 with full call graphs'`
- `docs/blog/posts/introducing-astix.md:259` — "10 Tier-1 languages"
- `docs/blog/posts/multi-language.md` (~L45) — "Tier 1 — 10 languages"
- `docs/public/llms.txt:10` — "38 languages across 3 tiers (10 Tier-1 with full call graphs)"
- `docs/docs/languages.md` (section header "Tier 1 — Full Intelligence (10 languages)" — to verify)

**Fix:** Replace "10 Tier-1" / "10 Tier 1" / "10 languages (Tier 1)" → "11 Tier-1" and add Swift + Dart to the list.

#### C5 — Sitemap leaks + missing `<lastmod>`
**File:** `dist/sitemap.xml` (21 URLs)
- Leaks: `/blog-ideas-astix` (cf. C3).
- All URLs missing `<lastmod>`, `<changefreq>`, `<priority>`.
- Trailing-slash inconsistency: `/blog/` vs `/blog/introducing-astix` (probably OK but review canonical alignment).

**Fix:** After C3, enhance VitePress sitemap config to emit `lastmod` from git:
```ts
sitemap: {
  hostname: 'https://astix.io',
  transformItems: (items) => items.map(i => ({ ...i, lastmod: new Date() })),
},
```
(or read from git log per page).

### 🟡 High (significant ranking/CTR impact — fix within 1 week)

#### H1 — Pricing page: dead-end + thin content
- Zero internal links (`pricing.md`).
- ~200 words, below 300-word thin-content threshold.
- Description `179 chars` (Google truncates at ~160).
**Fix:** Shorten description to ~150 chars. Add "Next steps" section linking to `/docs/getting-started`, `/docs/mcp-tools`, `/blog/introducing-astix`. Add intro paragraph defining astix (`pricing.md` opens with "Simple, transparent pricing" — needs a product definition sentence).

#### H2 — Title tags > 70 chars on 4/5 key pages
| Page | Current | Chars | Target |
|------|---------|------:|--------|
| index.md | "astix — Semantic code intelligence for AI coding assistants" | 71 | "astix — Code intelligence for AI coding assistants" (53) |
| pricing.md | (to verify, 74) | 74 | "Pricing — astix" or "astix pricing: Free, Solo, Team, Enterprise" (~55) |
| mcp-tools.md | (67) | 67 | "MCP tools reference — astix" (~38) |
| languages.md | (67) | 67 | "Supported languages — astix" (~35) |

VitePress auto-appends `| astix` (`siteTitle: false` but `title: 'astix'`). Consider setting explicit `titleTemplate: ':title'` to avoid the `| astix` duplication on pages whose title already contains "astix".

#### H3 — Description > 170 chars
- `pricing.md`: 179 chars.
- `mcp-tools.md`: 171 chars.
**Fix:** Trim to 140-160.

#### H4 — Zero internal linking on 4/5 key pages
`pricing.md`, `getting-started.md`, `mcp-tools.md`, `languages.md` have no cross-links. Add 3-5 contextual links per page.

#### H5 — Content gaps for AI search
Missing standalone definition/explanation articles:
- "What is astix?" (150-200 words factual definition).
- "What is MCP (Model Context Protocol)?"
- "What is semantic code intelligence?"
- "astix vs Sourcegraph / Semgrep / grep" (comparison article).
- "How does astix work?" (tree-sitter → PostgreSQL → pgvector → MCP).

**Impact:** Currently pricing.md and getting-started.md have <3 "citable passages" (short, factual, self-contained, quotable). AI crawlers need these to extract citations for AI Overviews / ChatGPT search / Perplexity.

#### H6 — FAQ answers too short (47% under 50 words)
`docs/.vitepress/data/faq.ts` has 17 items, 8 of them < 50 words. Google's FAQPage rich-results guidance favors 40-300 word answers. Expand the 8 short ones.

#### H7 — Schema.ts: Team pricing monthly missing from Product.offers
Offers array lists Team annual ($299/mo + $29/seat) but FAQ and PricingMatrix mention Team monthly ($359/mo + $39/seat). Add to schema.

#### H8 — Home missing FAQPage + HowTo schemas
`index.md` could benefit from:
- `FAQPage` with 3-5 top conceptual Q&A (shares JSON-LD footprint with `/pricing`).
- `HowTo` for the 5-step quick-start (`npx @astix/mcp-server init`, etc.).

### 🟠 Medium (optimization — within 1 month)

#### M1 — og:title brand-name duplication
`og:title: "astix — Semantic code intelligence for AI coding assistants | astix"` — VitePress appends `| astix`. Fix: config.ts `titleTemplate: false` or set `titleTemplate: ':title'` where page title already contains the brand.

#### M2 — Performance: heavy client bundles
- `mermaid.core.*.js` = 595 KB (loaded on blog posts with diagrams).
- `PricingMatrix.vue` = 1288 LOC (single chunk).
- `HeroShader.vue` = WebGL Canvas, 70 nodes, RAF loop — no `prefers-reduced-motion` guard.
- Total mermaid ecosystem ~1.7 MB when a diagram-heavy page is visited.

**Fix:**
- Code-split PricingMatrix via dynamic import.
- Gate HeroShader on `prefers-reduced-motion: no-preference` + lower mobile node count.
- Audit which pages actually need mermaid — if only blog posts, route-split is already done. If any docs/ page imports it at top, move to `<ClientOnly>` lazy.

#### M3 — Missing schemas to generate
| Schema | Location | Impact |
|--------|----------|--------|
| `HowTo` | home + getting-started | +CTR on "how to" queries |
| `BreadcrumbList` | all /docs/* and /blog/* | Rich sitelinks in SERP |
| `VideoObject` | home (demo video TODO L209) | Video rich result |
| `SoftwareSourceCode` | home (codeRepository: GitHub) | "Open source" panel |

#### M4 — Blog post images without alt/width/height/lazy
Blog posts embed Unsplash images (`https://images.unsplash.com/photo-...?w=800&h=400`) without alt text, explicit dimensions, or lazy loading. Impact: CLS, accessibility, image SEO.

**Fix:** Replace inline `<img>` with `<img src="..." alt="Descriptive text" width="800" height="400" loading="lazy" decoding="async">`. Or move to local optimized assets under `docs/public/blog/`.

#### M5 — Author attribution on blog posts weak
Blog posts have "Olivier Orabona" name in frontmatter/content but no link to `/blog/authors/olivier-orabona`. BlogPostLayout.vue could render a bio card at post end (auto-link on author field).

#### M6 — No analytics / RUM
Zero web analytics — cannot measure real Core Web Vitals (LCP/INP/CLS from field data) or CTR from search. Options: Cloudflare Web Analytics (free, no cookies, GDPR-friendly), Plausible, Fathom. Coordinate with existing TODO analytics task (line 157) about ePrivacy/CNIL.

#### M7 — Missing `font-display: swap`
Inter/IBM Plex Sans loaded via `@fontsource-variable` — verify `font-display: swap` is set (default may be `block` causing FOIT).
